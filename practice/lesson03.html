  • ECMAScript

    もともとJavaScriptはNetscape社のブラウザで使われる言語として誕生した。
    他のブラウザでの使用は想定されておらず、各ブラウザはJavaScriptを独自に拡張していた。
    JavaScriptはまったく標準化がされていない言語だった。

    その結果いろいろと問題が発生したため（ブラウザ毎で動作したりしなかったり、予期しない動作をしたり。）、
    Netscape社は国際標準化団体のECMA International※に依頼し、JavaScriptのコア部分を標準化した。
    これをECMAScriptと呼ぶ。
    JavaScriptはECMAscriptに準拠した言語。

    ※Ecma InternationalはECMAScript以外にもC#やDartなどの標準化もしている。

  • バージョンの表記について

    ほぼ全部のブラウザで動くのがES5で今までと同じ書き方。
    いま標準になっているのがES6で新しい機能や書き方。
    2015年以降は毎年新バージョンがリリースされ、
    年号をバージョン名としている。

    バージョン 公開日
    ECMAScript1 1997年6月
    ECMAScript2 1998年6月
    ECMAScript3 1999年12月
    ECMAScript4 破棄
    ECMAScript5 2009年12月
    ECMAScript5.1 2011年6月
    ECMAScript2015(ES6とも呼ばれてる) 2015年6月
    ECMAScript2016 2016年6月

  • ES5とES2015(ES6)のバージョンでの違い

    ES2015ではより書きやすく便利な文法が追加されたり、
    便利な標準ライブラリが追加されたりしている。

    let/constによる変数/定数の宣言

    ES5での変数宣言はvar一つでだったが、
    ES2015から変数の中身が変わるものはletで
    中身が変わらないものはconstで変数宣言する。

    ES5

    
  var test = 'テスト';
  for( var count = 0; count < 10; count++ ){
    console.log(test+count);
  }

    ES2015

    
  // こちらは定数（固定値）のためconstを使う
  const test = 'テスト';
  // こちらは変数の値が変わるのでletを使う
  for( let count = 0; count < 10; count++ ){
    console.log(test+count);
  }

    アロー関数

    ES5

    
  var test = function(){
    console.log('test');
  }
  test();

    ES2015

    
  const test = () => {
    console.log('test');
  }
  test();

    テンプレート文字列(文字列の結合)

    ES5

    
  var sei = '鈴木';
  var mei = '一郎';
  console.log('私の名前は'+sei+mei+'です。');

    ES2015

    
  const sei = '鈴木';
  const mei = '一郎';
  // 'シングルクォート'や"ダブルクォート"ではなく`バッククォート`を使う
  console.log(`私の名前は${sei}${mei}です。`);
