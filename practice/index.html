  • デバッグ方法

    console.log()

    ウェブコンソールに単一の文字列や変数の値などを出力する機能。

              
  console.log(変数名); // 変数の中身を表示する
  例：
  var name = "Yamada";
  console.log(name); // デバッガーツールのconsoleタブにYamadaが表示される

  console.log(文字列); // 文字列を表示する
  例：
  console.log("山田"); // デバッガーツールのconsoleタブに山田が表示される

    alert()

    console.logとほぼ同じだが、内容をダイアログで出力する。あまり使わない。

              
  alert(変数名); // 変数の中身を表示する
  例：
  var name = "Yamada";
  alert(name); // ダイアログにYamadaが表示される

  alert(文字列); // 文字列を表示する
  例：
  alert("山田"); // ダイアログに山田が表示される

  • 変数 - variable -

    文字列や数値などのデータに名前をつけ繰り返し利用できるようにする機能。
    データを入れておく箱をイメージするとわかりやすい。 

              
  // 変数宣言
  var name = "Yamada";
  console.log(name); // Yamadaが表示される
  // 動的に値を変更できる。再代入という。
  name = "Suzuki";
  console.log(name); // Suzukiが表示される

  // 変数名だけ用意し、後から値を入れる事もできる。
  // 変数名をカンマで繋ぐ良く使われる書き方
  var firsrName, lastName;
  firstName = "Yamada";
  lastName = "Tarou";

  // Bad pattern
  // 変数宣言しないとグローバル変数となり、
  // 再代入と混同され予期せぬエラーの原因となったり、
  // "use strict"モード（厳格モード）でエラーになるので必ず変数宣言をする。
  firstName = "Satou";

    無効な書き方

              
  var 1name = "ohori"; // 数字から始まっている
  var var = "ohori"; // 予約後は変数名に使えない 予約後一覧 https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Reserved_Words
  var first name = "ohori"; // スペースは使えない
  var name = ohori; // 文字列はシングルクォーテーションかダブルクォーテーションで囲う
  var name = 'ohori"; // シングルクォーテーションとダブルクォーテーションを混在させない
  var name = "ohori'; // シングルクォーテーションとダブルクォーテーションを混在させない

    有効な書き方

              
  var name = "Yamada"; // 小文字 alphabet
  var NAME = "Yamada"; // 大文字 alphabet
  var $name = "Yamada"; // $
  var _name = "Yamada"; // _
  /**
   * 数字や真偽値（true, false）などはクォーテーションはいらない。
   * クォーテーションで囲うと文字列として扱われる。
   */
  var num = 1; // 数字 - Number -
  var num = "1"; // 文字列 - String -
  var flag = true; // 真偽値  - Boolean -
  var flag = "true"; // 文字列 - String -
  var value = undefined; // Undefined
  var value = "undefined"; // 文字列 - String -
  var box = null; // Null
  var box = "null"; // 文字列 - String -

    より良い書き方 - Best practice -

              
  var c = "Yamada"; // Bad 変数が何を意味するかわからない。前提として他の人が見ても理解できるように書く。
  var first_name = "Yamada"; // Bad snake_case 文字列と関数名はキャメルケースの方が良い
  var FIRST_NAME = "Yamada"; // Bad 大文字のみの場合は定数の場合に使う

  var firstName = "Yamada"; // Good camelCase 文字列のほか関数名に使う

    メモ

    [ googleのガイドライン命名規則 ]

    • 名前には半角英数字とアンダースコア「_」とドルマーク「$」を使用する。
    • 外部(プログラムを書いた人以外)の人が見ても分かりやすい名前にする。
    • 独自のルールで単語の文字を省略したりしない。 
      悪い例：「wgcConnections」。// 何を指しているか外部の人には意味不明
    • クラス名は先頭大文字のキャメルケース(複数語を単語の先頭を大文字で結合したもの)。 
      例：BigDog
    • 関数名は先頭小文字のキャメルケース。 
      例：sendMessage
    • 定数名はすべて大文字のスネークケース(複数語をアンダースコア「_」で結合したもの）。 
      例：COLOR_CODE_RED
    • 変数名(引数名)は先頭小文字のキャメルケース。 
      例：priceCountReader

    googleのガイドライン：https://google.github.io/styleguide/jsguide.html#naming

  • 演算子 - operator -

    2 + 3の+のように、演算を表す記号のこと。
    このとき2や3は被演算子（ひえんざんし、operand、オペランド）という。

    算術演算子

    足し算、引き算、掛け算、割り算などで使われる演算子

              
  // 加算演算子
  1 + 2;
  // 減算演算子
  2 - 1;
  // 乗算演算子
  2 * 2;
  // 除算演算子
  4 / 2;
  // 基本左から順に計算されるが、加算と減算より乗算と除算が優先して計算される
  4 + 4 / 2 // => 6
  // 加算と減算を先に計算したい場合()で囲むとそちらが優先される
  (4 + 4) / 2 // => 4

    代入演算子

    変数に値を代入し、代入した値を返す。

              
  // 代入
  var num = 10;
  // 加算代入 変数numに1を足し再代入
  num += 1; // num = num + 1　と同じ;
  // 減算代入 変数numに1を引き再代入
  num -= 1; // num = num - 1　と同じ;
  // 乗算代入 変数numに2をかけ再代入
  num *= 2; // num = num * 2　と同じ;
  // 除算代入 変数numに2を割り再代入
  num /= 2; // num = num / 1　と同じ;

    比較演算子

    比較演算子はオペランド同士の値を比較し、真偽値(true, false)を返す演算子。

    厳密等価演算子（===）

    同じ型で同じ値である場合に、trueを返す。

              
  1 === 1; // => 同じ値と型なので true
  "1" === "1"; // => 同じ値と型なので true
  1 === "1"; // => 同じ値だが型が違うので false

    等価演算子（==）

    同じ値であれば同じ型でなくても、trueを返す。
    ※意図しない挙動となることがあるため厳密等価演算子（===）を使う。

              
  1 == 1; // => 同じ値と型なので true
  1 == "1"; // => 型は違うが同じ値なので true

    厳密不等価演算子（!==）

    値と型どちらかが等しくない時trueを返す。

              
  1 !== 1; // => 同じ値と型なので false
  1 !== "1"; // => 同じ値で型が等しくないので true

    不等価演算子（!=）

    値が等しくない時trueを返す。
    ※意図しない挙動となることがあるため厳密不等価演算子（!==）を使う。

              
  1 != 1; // => 同じ値と型なので false
  1 != "1"; // => 型は違うが同じ値なので false
  1 != 2; // => 値が違うので true

    大なり演算子/より大きい（>）

    左辺が右辺より大きい時、trueを返す。

              
  2 > 1; // => true
  2 > 2; // => false

    大なりイコール演算子/以上（>=）

    左辺が右辺より大きいまたは等しい時、trueを返す。

              
  2 >= 1; // => true
  2 >= 2; // => true
  2 >= 3; // => false

    小なり演算子/より小さい（<）

    左辺が右辺より小さい時、trueを返す。

              
  1 < 2; // => true
  2 < 1; // => false
  2 < 2; // => false

    小なりイコール演算子/以下（<=）

    左辺が右辺より小さいまたは等しい時、trueを返す。

              
  1 <= 2; // => true
  2 <= 2; // => true
  3 <= 2; // => false

    論理演算

    左右の真偽知(true, false)の組み合わせによって真偽知(true, false)を返す。

    AND演算子（&&）

    左辺及び右辺の値が共にtrueだった場合trueを返す。

              
  var x = true;
  var y = false;
  var z = true;
  x && y; // => false
  x && z; // => true

    OR演算子（||）

    左辺及び右辺のどちらか1つでもtrueだった場合trueを返す。

              
  var x = true;
  var y = false;
  var z = false;
  x || y; // => true
  y || z; // => false

  • 条件分岐

    「もし～だったら～する」というように条件を満たすかどうかで処理を変える。

    if文

    ()内の条件式が成り立つ場合に{}内の処理を実行する。
    条件式には真偽値(true, false)以外の値も指定できる。
    真偽値以外の場合、その値は暗黙的に真偽値(true, false)へ変換され判定される。
    以下リスト以外の値はtrueとして判定される。

    • false
    • undefined
    • null
    • 0
    • NaN
    • ""（空文字列）
              
  if ("条件式") {
    alert("条件式が成り立った場合処理を実行");
  }

  var x = 10;
  if(x > 9){ // 条件式がtrueなので実行される。
    alert(x);
  }

  var name = "Yamada";
  if(name){ // 条件式がtrueとして判定される。
    alert("実行");
  }

  var name = undefined;
  if(name){ // 条件式がfalseとして判定される。
    alert("実行");
  }

    else if文

    複数の条件分岐を書く場合は、if文に続けてelse if文を使うことで書ける。

              
  var num = 2;
  if(num === 1){
    console.log(1);
  } else if(num === 2){
    console.log(2);
  } else if(num === 3){
    console.log(3);
  }

    else文

    条件に一致しなかった場合の処理は、else文を使うことで書ける。

              
  var num = 2;
  if(num === 1){
    console.log(1);
  } else{
    console.log("numが1以外だった時は全て実行");
  }

    switch文

    式の評価結果が指定した値である場合に処理を行う。
    条件が複数ある場合に使用する事が多い。

              
  switch (式) {
    case ラベル1:
      // `式`の評価結果が`ラベル1`と一致する場合に実行する文
      break;
    case ラベル2:
      // `式`の評価結果が`ラベル2`と一致する場合に実行する文
      break;
    default:
      // どのcaseにも該当しない場合の処理
      break;
  }

  var num = 2;
  switch (num) {
    case 1:
        console.log(1);
        break;
    case 2: // 一致するケース
        console.log(2);
        break;
    case 3:
        console.log(3);
        break;
    default:
        console.log("default");
        break;
  }

  • 関数 - function -

    関数とは、ある一連の手続き（文の集まり）を1つの処理としてまとめる機能。
    関数を利用することで、同じ処理を毎回書くのではなく、一度定義した関数を呼び出すことで同じ処理を実行できる。

    関数宣言で書く方式

    function + 半角スペース + 関数名(){}
    実行が始まるまでに生成されるため、関数宣言の手前でも呼び出せる。

              
  testFunc(); // エラーにならない。
  function testFunc() {
    var functionName = "Test";
    console.log(functionName);
  }
  testFunc(); // エラーにならない。

    関数式で書く方式

    var 変数名 = function(){}
    こちらは関数名がないので無名関数とも言われる。
    宣言されるまで生成されないので、関数宣言の手前では呼び出せない。

    
  sampleFunc(); // エラーになる。
  var sampleFunc = function() {
    var functionName = "Sample";
    console.log(functionName);
  }
  sampleFunc(); // エラーにならない。

    即時関数

    通常の関数はその関数を呼び出すまで実行されないが、即時関数は呼び出さずに実行できる。
    一箇所しか使わない定義に関しては即時関数を使ってシンプルに書いた方が良い。

              
(function(){
  console.log("即時関数");
})();

// 変数宣言と一緒にこのようにかける。
var total = (function(){
  return (2 + 5); // 関数の結果を変数名totalに代入
})();
console.log(total); // => 7

    グローバル変数、ローカル変数

    「グローバル変数」はプログラムのどこからでもアクセスができる変数
    「ローカル変数」は関数の内側で宣言した変数

              
  var globalName = "Yamada"; // グローバル変数
  function sampleFn(){
    var localName = "Tanaka"; // ローカル変数
  }

  console.log(globalName); // グローバル変数:Yamadaが表示される
  console.log(localName); // ローカル変数:エラーになる

  function sampleFn2(){
    var localName2 = "Suzuki"; // ローカル変数
    console.log(globalName); // =>Yamada: グローバル変数
    console.log(localName2); // =>Tanaka: ローカル変数
    console.log(localName); // => error: ローカル変数
  }

  sampleFn2(); // 関数呼び出し

    引数 - argument -

    関数呼び出し時に渡す値

              
  function argFunc(name){
    console.log(name);
  }
  argFunc('arg');

  //デフォルト引数
  function defaultArgFunc(name = "Suzuki"){
    console.log(name);
  }
  defaultArgFunc('Yamada'); // Yamadaが表示される
  defaultArgFunc(); // デフォルト引数のSuzukiが表示される
